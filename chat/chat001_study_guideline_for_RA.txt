  Part 1: Understanding the Paper's Problem Formulation (Section III)

  What the Paper Does

  The paper addresses a hybrid beamforming design problem for multi-user massive
   MIMO-OFDM THz systems. Let me break down the key mathematical elements:

  System Variables:
  - W - Analog beamformer (Nt × NRF) with constant modulus constraint
  - F[k] - Digital beamformer (NRF × U) for each subcarrier k
  - H[k] - Channel matrix (U × Nt) for subcarrier k

  The SINR Expression:
                  |S₀|² |Hq[k] W Fq[k]|²
  SINRq,k = ─────────────────────────────────────
            Inter-user interference + ICI + Noise

  The Optimization Problem (Original):
  MAXIMIZE:  SE = Σq Σk log₂(1 + SINRq,k)

  SUBJECT TO:
    |(W)m,n| = 1/√Nt    (Constant modulus - makes it NON-CONVEX)
    ||WFq[k]||²F = 1    (Power normalization)

  Why It's Difficult:
  1. The constant modulus constraint |wij| = constant is non-convex
  2. W and F are coupled in both objective and constraints
  3. The log-SINR function with interference is non-convex

  Objective 1: Maximize Total Sum Rate

  ┌─────────────────────────────────────────────────────────┐
  │  MAXIMIZE:   Σ(q=1 to U) Σ(k=1 to K) log₂(1 + SINRq,k) │
  │                                                         │
  │  SUBJECT TO:                                            │
  │    • Total power constraint: Σ||wq||² ≤ Ptotal         │
  │    • Constant modulus (if hybrid): |wij| = 1/√Nt       │
  │    • Per-user power: ||WFq||² ≤ Pmax                   │
  └─────────────────────────────────────────────────────────┘

  When to use: When you want to maximize overall system throughput (total data
  rate)

  Solution approaches:
  - WMMSE (Weighted Minimum Mean Square Error)
  - Alternating Optimization
  - Successive Convex Approximation (SCA)

  Level 2: Optimization Theory (Core Knowledge)

  2.1 Convex Optimization (Most Important)

  Study Order:
  1. Convex sets and functions - Recognize what's convex
  2. Standard forms - LP, QP, SOCP, SDP
  3. Duality theory - Lagrangian, dual problem
  4. KKT conditions - Optimality conditions
  5. Algorithms - Gradient descent, interior-point

  Key Concepts for Wireless:

  CONVEX PROBLEM (Easy to solve globally):
     minimize    f(x)           ← convex function
     subject to  gi(x) ≤ 0      ← convex constraints
                 Ax = b         ← affine equality

  NON-CONVEX PROBLEM (Hard - local optima):
     - Log-SINR with interference
     - Constant modulus |x| = c
     - Rank constraints

  Primary Resource: https://web.stanford.edu/~boyd/cvxbook/ (Free PDF and video
  lectures)

  2.3 Lagrangian Duality & KKT Conditions

  Why Critical: These are used EVERYWHERE in wireless optimization.

  The Lagrangian:
  L(x, λ, ν) = f(x) + Σi λi·gi(x) + Σj νj·hj(x)

  where:
    f(x) = objective function
    gi(x) ≤ 0 = inequality constraints
    hj(x) = 0 = equality constraints
    λi ≥ 0 = dual variables (inequality)
    νj = dual variables (equality)

  KKT Conditions (Necessary for optimality):
  1. Stationarity:     ∇f(x*) + Σλi*∇gi(x*) + Σνj*∇hj(x*) = 0
  2. Primal feasibility: gi(x*) ≤ 0, hj(x*) = 0
  3. Dual feasibility:   λi* ≥ 0
  4. Complementary slackness: λi*·gi(x*) = 0

  Resource:
  https://www.stat.cmu.edu/~ryantibs/convexopt-S15/scribes/12-kkt-scribed.pdf

  3.3 Beamforming Types

  DIGITAL BEAMFORMING (Full flexibility, expensive):
    - One RF chain per antenna
    - Can control both amplitude and phase

  ANALOG BEAMFORMING (Limited, cheap):
    - Phase shifters only

  HYBRID BEAMFORMING (Compromise):
    - Analog (W) + Digital (F) cascade
    - Fewer RF chains than antennas

  Level 4: Specific Algorithms You Must Implement

  4.1 Zero-Forcing (ZF) Precoding

  Concept: Eliminate inter-user interference completely

  Formula:
  # Effective channel
  He = H @ W  # After analog beamformer

  # ZF precoder (pseudo-inverse)
  F_zf = He.conj().T @ inv(He @ He.conj().T)

  # Normalization
  for each user u:
      F[:,u] = F_zf[:,u] / norm(W @ F_zf[:,u])

  Properties:
  - Simple, closed-form solution
  - Suboptimal (noise enhancement)
  - Works well at high SNR

  4.2 MMSE Precoding

  Concept: Balance interference suppression with noise enhancement

  Formula:
  F_mmse = Hᴴ(HHᴴ + (σ²/P)I)⁻¹

  4.3 WMMSE Algorithm

  Key Insight: Maximizing sum-rate is equivalent to minimizing weighted MSE

  Rate maximization ⟺ Weighted MSE minimization

  ALGORITHM:
  1. Initialize beamformers W, F
  2. REPEAT:
     a. Update receive filters: uq = MMSE receiver
     b. Update MSE weights: wq = 1/MSEq
     c. Update transmit beamformers: solve weighted MMSE
  3. UNTIL convergence

  4.4 Manifold Optimization (for Constant Modulus)

  Why needed: Constant modulus constraint |wij| = 1/√Nt is non-convex

  Key Concepts:
  MANIFOLD: The set of all valid solutions
  M = {x ∈ ℂⁿ : |xi| = c, ∀i}  ← Complex circle manifold

  OPERATIONS:
  1. Riemannian Gradient: Project Euclidean gradient onto tangent space
  2. Retraction: Project updated point back onto manifold

  RETRACTION for constant modulus:
  R(x) = c · x / |x|  (normalize each element)

  Resources:
  - https://pymanopt.org/ (Python)
  - https://www.manopt.org/ (MATLAB)

  Part 6: Quick Reference - Problem Formulation Template

  When your professor gives you a new problem, use this template:

  STEP 1: IDENTIFY THE OBJECTIVE
    □ What to maximize/minimize?
    □ Sum rate? Power? Fairness? EE?

  STEP 2: IDENTIFY VARIABLES
    □ Beamforming vectors/matrices
    □ Power allocation
    □ User scheduling

  STEP 3: WRITE CONSTRAINTS
    □ Power constraints (total, per-antenna, per-user)
    □ SINR/rate requirements
    □ Hardware constraints (constant modulus, finite resolution)

  STEP 4: CHECK CONVEXITY
    □ Is objective convex/concave?
    □ Are constraints convex?
    □ If yes → Use CVX directly
    □ If no → Choose appropriate technique

  STEP 5: SELECT ALGORITHM
    □ Non-convex → Alternating opt, SCA, SDR
    □ Constant modulus → Manifold optimization
    □ Rate maximization → WMMSE
    □ Power minimization with SINR → SDR or duality

  STEP 6: IMPLEMENT AND VALIDATE
    □ Start with simple case
    □ Verify KKT conditions
    □ Compare with benchmarks
